# Deploy sockuraaa to Netlify

## Overview
Deploy the Vite + React 18 SPA to Netlify. The project outputs a static `dist/` folder after `tsc -b && vite build`. Netlify supports SPA routing via a `_redirects` file or `netlify.toml`.

## Files to create

### 1. `netlify.toml` (project root)
```toml
[build]
  command = "npm run build"
  publish = "dist"

[[redirects]]
  from = "/*"
  to = "/index.html"
  status = 200
```

### 2. `public/_redirects` (project root)
Create `public/_redirects` with:
```
/*    /index.html   200
```
This ensures client-side routing (React Router) works correctly on Netlify.

## Deployment steps
1. Commit and push the above files to GitHub
2. Connect the GitHub repo to Netlify via the Netlify dashboard (or `netlify deploy` CLI)
3. Netlify auto-detects the `netlify.toml` and builds/deploys

## Verification
- Run `npm run build` locally and confirm `dist/` is generated
- After deploying, navigate to a sub-route (e.g., `/about`) and confirm it loads `index.html` (not a 404)
